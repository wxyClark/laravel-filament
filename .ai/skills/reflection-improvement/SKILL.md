# Reflection & Improvement — 反思与提升

> **目标**: 通过系统化反思，持续优化 AI Agent 的交互质量和工作效率
> **频率**: 每个重要任务完成后，或用户主动要求
> **输出**: 回顾文档 + 改进计划

---

## 核心原则

```
📊 数据驱动 → 基于实际数据而非主观感受
🎯 聚焦改进 → 识别可操作的改进点
📈 持续优化 → 建立反馈循环
📝 知识沉淀 → 将经验转化为可复用的知识
```

---

## 反思流程

### Phase 1: 数据收集

```bash
# 1. 收集代码变更
git diff --stat HEAD~5..HEAD

# 2. 收集测试结果
./vendor/bin/pest --compact

# 3. 收集代码质量
./vendor/bin/pint --test
./vendor/bin/phpstan analyse --no-progress

# 4. 收集提交历史
git log --oneline -20
```

### Phase 2: 分析评估

#### 2.1 交互质量分析

| 维度 | 评估标准 | 评分 |
|------|----------|------|
| 理解准确性 | 是否正确理解用户意图 | 1-5 |
| 响应及时性 | 是否快速响应 | 1-5 |
| 解决有效性 | 是否有效解决问题 | 1-5 |
| 沟通清晰度 | 是否清晰表达 | 1-5 |

#### 2.2 技术质量分析

| 维度 | 评估标准 | 评分 |
|------|----------|------|
| 代码质量 | Pint + PHPStan 通过率 | 1-5 |
| 测试覆盖 | 测试覆盖率 | 1-5 |
| 架构合规 | DDD 分层是否正确 | 1-5 |
| 性能影响 | 是否引入性能问题 | 1-5 |

#### 2.3 流程效率分析

| 维度 | 评估标准 | 评分 |
|------|----------|------|
| 步骤优化 | 是否有冗余步骤 | 1-5 |
| 工具使用 | 是否高效使用工具 | 1-5 |
| 知识复用 | 是否复用已有知识 | 1-5 |

## 度量指标

### 交互效率

| 指标 | 计算方式 | 目标值 |
|------|----------|--------|
| 理解准确率 | 正确理解次数 / 总次数 | > 90% |
| 一次成功率 | 一次解决次数 / 总次数 | > 80% |
| 平均响应时间 | 总响应时间 / 次数 | < 5s |

### 技术质量

| 指标 | 计算方式 | 目标值 |
|------|----------|--------|
| 代码通过率 | Pint+PHPStan 通过次数 / 总次数 | 100% |
| 测试通过率 | 测试通过次数 / 总次数 | 100% |
| 架构合规率 | 架构测试通过次数 / 总次数 | 100% |

### 知识复用

| 指标 | 计算方式 | 目标值 |
|------|----------|--------|
| 知识库条目数 | 累计条目数 | 持续增长 |
| 复用率 | 复用次数 / 总次数 | > 60% |
| 更新频率 | 月更新次数 | > 5次 |

## 最佳实践

1. **及时反思**: 任务完成后立即进行反思
2. **数据驱动**: 基于实际数据而非主观感受
3. **聚焦改进**: 识别可操作的改进点
4. **知识沉淀**: 将经验转化为可复用的知识
5. **持续优化**: 建立反馈循环
6. **文档化**: 所有反思和改进都要文档化
7. **分享经验**: 与团队分享有价值的经验
