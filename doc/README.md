# 文档目录结构

## 目录说明

```
doc/
├── requirements/          # 需求分析文档
│   └── TEMPLATE.md       # 需求文档模板
├── design/               # 设计文档
│   └── TEMPLATE.md       # 设计文档模板
├── development/          # 开发文档
│   └── TEMPLATE.md       # 开发文档模板
├── testing/              # 测试文档
│   └── TEMPLATE.md       # 测试文档模板
├── deployment/           # 部署文档
│   └── TEMPLATE.md       # 部署文档模板
├── retrospective/        # 回顾文档
│   └── TEMPLATE.md       # 回顾文档模板
├── PRD/                  # 产品需求文档 (已有)
├── BestPractice/         # 最佳实践 (已有)
├── Core/                 # 核心文档 (已有)
└── README.md             # 本文件
```

## 文档编号规则

| 类型 | 前缀 | 格式 | 示例 |
|------|------|------|------|
| 需求文档 | REQ | REQ-{模块}-{序号} | REQ-USER-001 |
| 设计文档 | DES | DES-{模块}-{序号} | DES-USER-001 |
| 开发文档 | DEV | DEV-{模块}-{序号} | DEV-USER-001 |
| 测试文档 | TES | TES-{模块}-{序号} | TES-USER-001 |
| 部署文档 | DEP | DEP-{模块}-{序号} | DEP-USER-001 |
| 回顾文档 | RET | RET-{模块}-{序号} | RET-USER-001 |

## 文档生命周期

```
草稿 → 评审中 → 已确认 → 已废弃
```

## 文档关联关系

```
REQ (需求) 
  ↓
DES (设计)
  ↓
DEV (开发) ← TES (测试)
  ↓
DEP (部署)
  ↓
RET (回顾)
  ↓
优化 REQ/DES/DEV/TES
```

## Git 追踪

所有文档都提交到 Git 仓库，用于：

1. **版本追踪**: 追踪文档变更历史
2. **变更审计**: 了解谁在什么时候修改了什么
3. **协作追踪**: 多人协作时了解各自贡献
4. **知识沉淀**: 团队知识积累

## 使用建议

1. **使用模板**: 新文档基于模板创建
2. **及时更新**: 代码变更时同步更新文档
3. **关联引用**: 文档之间相互引用
4. **定期回顾**: 每个迭代结束后进行回顾
5. **持续改进**: 根据回顾结果优化流程
