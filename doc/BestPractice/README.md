## 🎯 核心理念

### 母版提示词系统

> 通过简单的母版提示词，让 AI 工具基于这些内容通过交互式问答的方式，自动组装精确的结构化的提示词。

### 工作原理

```
用户输入需求 → AI 读取母版提示词 → AI 提问引导 → 用户回答 → AI 自动组装提示词 → 执行任务
```

## 🔄 五种使用模式

### 模式一：基于 PRD 快速开发

**适用场景**: 项目启动初期，已有 PRD 文档

**母版提示词**:
```
请基于 doc/PRD/{module}/ 下的文档，为用户故事 {story_id} 生成代码。

要求：
1. 读取对应的故事、模型、API 文档
2. 生成完整的代码实现
3. 包含单元测试
```

**详细指南**: [01-prd-to-code.md](./01-prd-to-code.md)

---

### 模式二：新需求的提示词设计

**适用场景**: 收到新需求后，快速定义需求并开发实现

**母版提示词**:
```
我有一个新需求：{需求描述}

请帮我：
1. 先分析需求，生成 PRD 文档
2. 基于 PRD 生成代码实现
3. 包含测试用例
```

**详细指南**: [02-new-requirement-workflow.md](./02-new-requirement-workflow.md)

---

### 模式三：P9 面试官模式

**适用场景**: 需要技术决策，希望深度思考

**母版提示词**:
```
我要实现 {功能描述}，请像 P9 面试官一样引导我：

1. 先问我业务背景和技术约束
2. 让我提供技术方案
3. 评估我的方案并给出优化建议
4. 对比多种方案的设计原因
```

**详细指南**: [05-p9-interviewer-mode.md](./05-p9-interviewer-mode.md)

---

### 模式四：需求分析模式

**适用场景**: 需求模糊，需要专业分析

**母版提示词**:
```
我收到一个需求：{需求描述}

请帮我进行专业的需求分析：
1. 讲解需求分析的专业知识（5W2H、用户故事等）
2. 通过提问引导我澄清需求
3. 输出结构化的 PRD 文档
```

**详细指南**: [06-requirement-analysis.md](./06-requirement-analysis.md)

---

### 模式五：Code Review 模式

**适用场景**: 代码已完成，需要质量审查

**母版提示词**:
```
请对以下代码进行 Code Review：

{代码内容}

要求：
1. 从设计、质量、性能、安全四个维度审查
2. 给出具体的改进建议
3. 对比优化前后的代码
```

**详细指南**: [07-code-review-mode.md](./07-code-review-mode.md)

## 📊 场景匹配指南

| 场景特征 | 推荐模式 | 预期耗时 |
|---------|---------|---------|
| 已有 PRD，需要写代码 | 模式一 | 短 |
| 新需求，需要从头开始 | 模式二 | 中 |
| 技术方案不确定 | 模式三 | 长 |
| 需求模糊，需要澄清 | 模式四 | 中 |
| 代码已完成，需要审查 | 模式五 | 中 |

## 🚀 快速开始

### 步骤 1：识别任务场景

根据你的任务特征，选择合适的模式。

### 步骤 2：使用母版提示词

将母版提示词复制到 AI 工具中。

### 步骤 3：交互式问答

按照 AI 的引导回答问题。

### 步骤 4：获取输出

AI 会自动组装精确的提示词并执行任务。
