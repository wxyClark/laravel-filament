## 🎯 两种最佳用法对比

| 维度 | 用法一：基于 PRD 快速开发 | 用法二：新需求的提示词设计 |
|------|------------------------|------------------------|
| **适用场景** | 项目启动初期，已有 PRD | 收到新需求后 |
| **输入** | 已生成的 PRD 文档 | 自然语言需求描述 |
| **输出** | 可用的系统代码 | PRD 文档 + 代码 |
| **核心流程** | PRD → 提示词 → 代码 | 需求 → PRD → 提示词 → 代码 |
| **关键技巧** | 引用 PRD 文档 | 渐进式生成 PRD |
| **推荐粒度** | 单个用户故事 | 单个功能模块 |
