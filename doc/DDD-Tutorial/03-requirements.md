# 03 需求梳理（用户故事 + 验收标准）

> 阶段目标：把头脑风暴收敛为可执行的需求条目，每条带验收标准（AC）。
> 角色：需求分析师。每个模块一份，作为 `06-modules/*.md` 的源头。

## 3.1 商品模块（Catalog）

- **US-C1** 运营可创建 SPU 并添加多个 SKU（不同规格）。
  - AC：SPU 含名称/类目/类型(实体|虚拟)；SKU 含规格键值、价格、库存、状态。
- **US-C2** 运营可创建套餐，引用多个 SKU 并设套餐价。
  - AC：套餐总价独立于组成 SKU 单价之和；下单时按组成 SKU 扣库存。
- **US-C3** 商品区分为实体/虚拟，影响后续履约策略。
  - AC：SPU 有 `type` 枚举；虚拟商品无物流，走码交付。

## 3.2 购物车模块（Cart）

- **US-CA1** 用户可将 SKU 加入购物车、改数量、选中/取消。
  - AC：同一 SKU 合并数量；超库存拦截。
- **US-CA2** 结算时购物车转为订单。
  - AC：仅选中项生成订单；生成后清空对应项。

## 3.3 订单模块（Order）

- **US-O1** 从购物车选中项生成订单，计算总额。
  - AC：含商品金额、运费（实体）、优惠；金额精度 decimal。
- **US-O2** 订单有状态机：待支付→已支付→已完成；可取消。
  - AC：状态迁移受约束，非法迁移抛异常。

## 3.4 支付模块（Payment）

- **US-P1** 用户可用余额支付订单。
  - AC：余额不足拦截；成功扣减并生成支付流水。
- **US-P2** 用户可充值余额。
  - AC：充值即增余额，生成充值流水。
- **US-P3** 支付方式可扩展（策略）。
  - AC：新支付网关只需实现 `PaymentGateway` 接口并注册。

## 3.5 售后模块（AfterSale）

- **US-A1** 用户可对已支付订单发起退款。
  - AC：退款回写余额/原路；关联原支付流水。
- **US-A2**（从简）退货退款：标记退货，库存回滚（实体）。
  - AC：虚拟商品不支持退货，仅退款。

## 3.6 履约发货（Fulfillment）

- **US-F1** 实体订单发货：录入物流单号。
  - AC：发货后订单状态→已完成（或待收货）。
- **US-F2** 虚拟订单发货：生成唯一码 + 密码并交付。
  - AC：码全局唯一；密码加密存储；用户可查看一次/重发。
