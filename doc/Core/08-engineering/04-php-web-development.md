# 🐘 PHP Web 开发全流程最佳实践

本章聚焦主流框架下的全流程最佳实践与大厂共识的行业规范，覆盖需求、架构设计、实现、测试、部署、运行运维、安全与合规等环节。重点结合主流框架生态，并新增对 Yii3 与 Hyperf 的实践要点。

## 适用范围
- 需求分析与架构设计的通用原则
- 实现层面的分层、接口约定、测试驱动开发
- 部署、监控、日志、CI/CD、自动化运维
- 安全、合规与密钥管理
- 针对 Yii3 与 Hyperf 的框架级实践

## 通用全流程要点
- 分层架构、接口定义、SOLID/DRY/KISS、依赖注入、参数化查询、输出转义、CSRF、密码哈希、环境变量配置、日志与监控、缓存与队列、静态分析等
- 统一的代码风格与 PSR 规范遵循（如 PSR-12、PSR-4）以及自动化代码质量工具
- 安全性：对接 OWASP Top 10、数据保护、最小权限、密钥轮换、密钥管理
- CI/CD：自动化测试、静态分析、镜像构建、分阶段环境（dev/stage/prod）的发布策略
- 可观测性：分布式追踪、指标、日志结构化、OpenTelemetry 集成
- 容器化与云原生：Docker/Kubernetes 最佳实践、无服务器组件的边缘场景
- 迁移与回滚策略：数据库迁移、版本化 API、蓝绿/增量发布

## Yii3 实践要点
- 概要：Yii3 是现代化的 PHP 框架，强调高效开发、模块化组件、良好的可测试性与社区生态。
- 架构与设计
  - 采用清晰的分层：模型/业务逻辑/持久化层分离，使用 DTO/VO 进行数据传输
- 实现要点
  - 控制器与路由：RESTful 风格或控制器方法映射，尽量使用资源路由与服务层解耦
  - 依赖注入：利用 Yii3 的依赖注入容器，解耦业务逻辑和实现
  - 数据访问：优先使用 Query Builder/ActiveRecord 的规范，避免暴露实现细节
- 安全与合规
  - 输入验证、输出转义、参数化查询、CSRF 防护、密码哈希
- 测试与质量
  - PHPUnit/Codeception 结合，覆盖单元、集成与端到端测试
- 部署与运维
  - 配置分环境变量、缓存策略、日志归档、性能监控
- 参考资源
  - Yii3 官方文档、PSR 规范、Composer、PHPUnit、PHPStan/Psalm

## Hyperf 实践要点
- 概要：Hyperf 基于 Swoole 的高性能框架，强调协程、事件驱动和微服务友好设计。
- 架构与设计
  - 使用 DI 容器和中间件，模块化服务划分，关注接口契约
- 实现要点
  - 异步编程：尽量编写协程友好的代码，使用协程安全的数据库驱动和中间件
  - 路由与控制器：支持注解/属性路由，清晰的控制器职责分离
  - 数据访问：Hyperc 的数据库组件、连接池、异步查询模式
- 安全与合规
  - 与其他框架一致的认证/授权、输入验证、输出转义、日志安全
- 测试与质量
  - PHPUnit、Hyperf 测试工具链，覆盖异步逻辑的测试用例
- 部署与运维
  - 基于 Swoole 的服务部署、水平扩展、容器化部署、健康检查和热重载
- 参考资源
  - Hyperf 官方文档、OpenTelemetry、Docker/Kubernetes、12 Factor 应用
