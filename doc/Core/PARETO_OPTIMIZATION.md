### 1. 数学思维方法论（07-science/04-mathematics-thinking.md）

**为什么重要？**
- 数学是**所有科学的基础语言**
- 数学思维培养**精确、逻辑、量化**的思考方式
- 在商业、技术、生活中无处不在

**核心方法（5个）**：

#### 📐 抽象化思维
**价值**：从具体问题中提取通用模式
```
应用场景：
- 产品设计：从用户需求抽象出通用功能
- 系统架构：从业务场景抽象出技术模型
- 问题解决：从具体案例提炼方法论
```

#### ✅ 证明思维
**价值**：严谨的逻辑推导和验证
```
应用场景：
- 代码审查：验证逻辑正确性
- 商业论证：检验假设和推理
- 决策制定：确保结论可靠
```

#### 📊 建模思维
**价值**：用数学描述现实，预测和优化
```
应用场景：
- 业务预测：销售、用户增长模型
- 资源优化：时间、预算分配
- 风险评估：概率和影响分析
```

#### 🎲 概率思维
**价值**：量化不确定性，做出理性决策
```
关键概念：
- 贝叶斯更新：根据新证据调整信念
- 期望值计算：长期平均结果
- 大数定律：短期随机，长期规律

应用场景：
- 投资决策：风险收益权衡
- A/B测试：统计显著性判断
- 医疗诊断：假阳性/假阴性
```

#### ⚡ 优化思维
**价值**：在约束下寻找最优解
```
应用场景：
- 算法设计：时间/空间复杂度优化
- 运营效率：流程、资源配置
- 个人管理：时间、精力分配
```

**跨界应用案例**：
- 商业定价策略（建模 + 优化）
- 个人时间管理（量化 + 优化）
- 产品组件库设计（抽象 + 模块化）

### 2. 心理学思维方法论（10-medicine/03-psychology-thinking.md）

**为什么重要？**
- 所有产品和服务最终都是**为人设计**
- 理解人类行为是**成功的关键**
- 心理学提供**科学的洞察工具**

**核心方法（5个）**：

#### 🧠 双系统思维
**价值**：理解直觉vs理性的运作机制
```
系统1（快思考）：
- 快速、自动、无意识
- 适合：专家直觉、熟悉情境
- 陷阱：认知偏差

系统2（慢思考）：
- 缓慢、费力、有意识
- 适合：重要决策、复杂问题
- 限制：容易疲劳

应用：
- 识别何时该信任直觉
- 设计决策流程触发系统2
- 优化 UI/UX 减少认知负荷
```

#### ⚠️ 认知偏差识别
**价值**：避免常见思维陷阱
```
高频偏差清单：
1. 确认偏差：只看到支持自己观点的证据
2. 可得性偏差：根据易回忆例子判断频率
3. 锚定效应：过度依赖首次信息
4. 损失厌恶：损失痛苦 > 收益快乐
5. 沉没成本谬误：因已投入而继续错误

防御工具：
- 检查清单
- 外部视角
- 事前验尸
- 红队演练
```

#### 🔥 动机分析框架
**价值**：理解和激发内在动力
```
自我决定理论（SDT）：
三大基本心理需求：
1. 自主性（Autonomy）：感到行为是自己选择的
2. 胜任感（Competence）：感到能够有效应对挑战
3. 归属感（Relatedness）：感到与他人连接

应用：
- 员工激励：满足三需求
- 产品设计：内在动机培养
- 教育改革：从外在奖励到内在兴趣
```

#### 🔄 行为改变技术
**价值**：科学地促进行为转变
```
COM-B 模型：
行为 = 能力（Capability）+ 机会（Opportunity）+ 动机（Motivation）

行为改变技巧（BCTs）：
1. 目标设定（SMART）
2. 行动计划（如果...那么...）
3. 反馈监控（追踪、可视化）
4. 社会支持（伙伴、社区）
5. 环境设计（Nudge）

应用：
- 健康习惯养成
- 用户激活和留存
- 组织变革管理
```

#### 💝 情绪智力（EQ）
**价值**：识别、理解、管理情绪
```
四大维度：
1. 自我觉察：识别自己的情绪
2. 自我管理：调节情绪和冲动
3. 社会觉察：共情和理解他人
4. 关系管理：影响力和冲突处理

应用：
- 领导力发展
- 团队协作
- 客户服务
- 谈判沟通
```

**跨界应用案例**：
- 产品设计：利用认知偏差和社会证明
- 团队管理：激发动机和心理安全感
- 个人决策：偏差管理和冷却期

## 🎯 使用建议

### 学习路径

**初学者**：
1. 先读"常见误区"部分，纠正错误认知
2. 选择 1-2 个最感兴趣的方法深入
3. 阅读"跨界应用"案例，理解实际价值

**实践者**：
1. 每周选择一个方法应用到实际工作
2. 记录应用过程和效果
3. 与团队分享和讨论

**进阶者**：
1. 结合多个方法解决复杂问题
2. 创建个人的思维框架和检查清单
3. 教导他人，深化理解

### 快速参考

**遇到问题时**：
- 需要量化分析？ → 数学思维（建模、概率、优化）
- 需要理解用户？ → 心理学（双系统、动机、偏差）
- 需要促进行为改变？ → 心理学（COM-B、习惯养成）
- 需要严谨论证？ → 数学思维（证明、逻辑）
