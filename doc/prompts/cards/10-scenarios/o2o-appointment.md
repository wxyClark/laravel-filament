# 📅 O2O 预约场景模板

> **版本**: v3.0 | **层級**: L5+ | **最后更新**: 2026-06-07

## 场景概述
O2O 预约系统的核心业务场景，包括时段管理、预约下单、核销退款等。

## 领域模型扩展

### 表结构
| 表名 | 说明 |
|------|------|
| `appointment_timeslots` | 预约时段 |
| `appointments` | 预约记录 |
| `services` | 服务项目 |
| `stores` | 门店 |

### 状态机
```
pending → confirmed → completed
           ↓           ↓
         cancelled ← cancelled
```

### 关键约束
- 时间片容量限制 + SQL 级并发锁
- 预约时间必须在服务营业时间内
- 退款需在预约开始前 N 小时

## 提示词模板
```
## L3: 角色
### 交易工程师 + O2O 领域约束

## L4: 任务
实现预约服务：
1. 时段查询（排除已满时段）
2. 预约创建（并发锁 + 容量校验）
3. 预约确认
4. 预约取消（退款）
5. 到店核销

## 领域约束
- constraint-o2o-timeslot-locking
```
