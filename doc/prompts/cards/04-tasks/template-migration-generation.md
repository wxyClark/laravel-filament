## 用途说明
标准化数据库表结构的创建过程。

## 适用场景
- 新功能开发初期的数据库建模。

## 标准内容块

```markdown
# 任务：创建 {table_name} 迁移文件

## 角色
@{Role}

## 要求
1. **字段类型**：根据数据类型选择最合适的 MySQL 字段（如金额用 `decimal(10,2)`）。
2. **索引优化**：为所有用于查询、排序和外键的字段添加索引。
3. **外键约束**：显式定义 `foreignId` 并设置 `onDelete('cascade')` 或 `restrict`。
4. **注释规范**：每个字段必须包含中文注释 (`->comment('...')`)。
