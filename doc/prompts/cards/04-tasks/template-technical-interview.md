# 任务模板：技术面试引导 (Technical Interview)

## 用途说明
在执行技术任务前，通过面试引导的方式，让开发者先思考方案，再评估优化。

## 适用场景
- 任何需要技术决策的开发任务
- 新功能设计前的技术评审
- 提升团队技术思维能力

## 标准流程

```markdown
# 技术面试引导流程

## 阶段 1：需求澄清
P9 面试官会问：
1. 这个需求的核心目标是什么？
2. 有哪些关键的业务约束？
3. 预期的用户量/并发量是多少？
4. 有哪些技术约束？

## 阶段 2：方案引导
P9 面试官会问：
1. 你打算如何实现这个功能？
2. 为什么选择这个方案而不是其他方案？
3. 这个方案的核心数据流是什么？
4. 如何处理边界情况和异常？

## 阶段 3：深度追问
P9 面试官会问：
1. 如果并发量增加10倍，你的方案还能支撑吗？
2. 这个方案的性能瓶颈在哪里？
3. 如何保证数据一致性？
4. 监控和报警如何设计？

## 阶段 4：方案评估
P9 面试官会：
1. 评估方案的正确性、性能、可维护性
2. 分析方案的优点和缺点
3. 对比多种方案的设计原因
4. 给出优化建议
```

## 输出格式

### 阶段 1-3：引导提问

```markdown
## 🎯 需求理解
{复述需求，确认理解正确}

## ❓ 引导问题

### 业务层面
1. {问题1}
2. {问题2}

### 技术层面
3. {问题3}
4. {问题4}

### 边界情况
5. {问题5}
6. {问题6}

## 💡 请先提供你的技术方案
在回答上述问题后，请给出你的技术方案，包括：
- 整体架构设计
- 核心数据模型
- 关键接口设计
- 异常处理策略

我会基于你的方案进行评估和优化建议。
```

### 阶段 4：方案评估

```markdown
## 📊 方案评估

### 优点分析
- ✅ {优点1}
- ✅ {优点2}

### 待优化点
- ⚠️ {待优化1}
- ⚠️ {待优化2}

### 优化建议
{详细的优化方案和设计原因}

### 方案对比
| 维度 | 你的方案 | 优化方案 | 说明 |
|------|---------|---------|------|
| 性能 | {评估} | {评估} | {说明} |
| 可维护性 | {评估} | {评估} | {说明} |
| 可扩展性 | {评估} | {评估} | {说明} |
```

## 评估维度

| 维度 | 权重 | 评估要点 |
|------|------|---------|
| **正确性** | 30% | 方案是否能满足需求 |
| **性能** | 25% | 方案的性能表现 |
| **可维护性** | 20% | 代码的可读性和可维护性 |
| **可扩展性** | 15% | 方案的扩展能力 |
| **安全性** | 10% | 方案的安全考虑 |
