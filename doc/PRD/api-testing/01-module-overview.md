# API 接口测试可视化系统 — 模块概述

> **文档编号**: PRD-ATS-001
> **版本**: v1.0
> **创建日期**: 2026-07-18
> **状态**: 已确认

---

## 1. 背景

当前 Laravel Filament 项目已有约 90 个 Pest 测试用例，但缺少一个可视化的接口测试管理平台。开发/测试人员需要：

- 手动配置接口参数并快速测试
- 管理测试环境（开发/测试/生产）
- 编排场景测试（多接口顺序执行 + 参数传递）
- 定时执行回归测试和冒烟测试
- 直观查看测试结果和历史趋势

---

## 2. 目标

| 目标 | 描述 |
|------|------|
| **可视化管理** | 按 模块→功能→接口 三层树形结构管理所有 API |
| **快速测试** | 为每个接口配置多组测试参数，一键执行 |
| **场景编排** | 支持多接口顺序执行，JSON Path 参数传递 |
| **环境切换** | 支持开发/测试/生产多环境配置 |
| **自动化测试** | 支持回归测试、冒烟测试、CI/CD 集成 |
| **结果可视化** | 仪表盘展示测试通过率、趋势、历史记录 |

---

## 3. 用户角色

| 角色 | 描述 | 核心操作 |
|------|------|----------|
| **开发者** | 开发接口时手动测试 | 配置参数、执行测试、查看结果 |
| **测试人员** | 编写测试用例、执行回归 | 管理用例、编排场景、执行计划 |
| **运维人员** | 监控接口健康状态 | 查看仪表盘、配置定时任务 |
| **项目经理** | 了解接口质量概况 | 查看测试报告、通过率趋势 |

## 5. 非功能需求

| 维度 | 要求 |
|------|------|
| **性能** | 单接口测试响应 < 5s，批量测试 < 30s |
| **并发** | 支持 10 个场景并行执行 |
| **存储** | 测试结果保留 90 天，支持清理 |
| **安全** | 敏感配置（API Key）加密存储 |
| **兼容** | 支持 RESTful API，预留 GraphQL 扩展 |

---

## 6. 技术约束

| 约束 | 说明 |
|------|------|
| **框架** | Laravel 12 + Filament 3.x |
| **数据库** | MySQL 8.0 (生产) / SQLite (测试) |
| **测试框架** | Pest PHP |
| **HTTP 客户端** | Laravel HTTP Client (Guzzle) |
| **JSON Path** | Flow JSONPath 或自实现 |
| **队列** | 用于异步执行长时间测试 |

---

## 7. 术语表

| 术语 | 定义 |
|------|------|
| **模块** | 业务功能分组，如"用户管理"、"订单管理" |
| **功能** | 模块下的二级分组，如"用户认证"、"用户信息" |
| **接口** | 具体的 API 端点，如 `POST /api/login` |
| **测试用例** | 一个接口的一组测试参数 + 断言 |
| **场景** | 多个接口的有序执行序列 |
| **测试计划** | 包含一个或多个场景的执行计划 |
| **环境** | 测试运行的目标环境（base_url + 认证配置） |
